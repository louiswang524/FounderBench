# KDD 2027 submission blockers

The technical draft and evidence package are ready for author review. Before submission, the owners still need to:

- replace author, affiliation, email, conference, and rights placeholders in `main.tex`;
- choose and publish the final repository license and citation metadata;
- confirm all authors have complete OpenReview profiles before the venue deadline;
- prepare the venue-required AI-assistance disclosure;
- check the final KDD 2027 call, template version, track name, and submission fields when released.

Recommended evidence improvements, but not required to read the current single-run study:

- repeat hosted runs to estimate model-sampling variance;
- obtain founder or expert calibration;
- instantiate a real private holdout rather than describing the visible `public_test` split as hidden;
- require exact API model identifiers at the run root for every future submission.
